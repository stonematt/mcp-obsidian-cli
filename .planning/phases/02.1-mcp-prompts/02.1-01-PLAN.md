
Adapt the content for an MCP AI assistant audience. The file should contain these sections:

1. **Title line** — `# Obsidian CLI Reference`
2. **Command Syntax** — How CLI commands are structured: `obsidian <command> [key=value...]`, parameter quoting rules, flag syntax
3. **Common Commands** — Reference table or list of the most useful commands: `read`, `search:context`, `daily:read`, `daily:append`, `create`, `tasks`, `tags`, `properties`, `property:set`, `property:read`, `backlinks`, `files`, `recents`, `help`
4. **Parameter Patterns** — How `file=`, `path=`, `query=`, `content=`, `name=`, `value=`, `limit=`, `sort=` work; quoting rules for values with spaces
5. **Vault Targeting** — How `--vault` works and when it is needed
6. **Output Behavior** — What CLI returns (plain text to stdout), stderr for errors
7. **Using This Knowledge with MCP Tools** — per D-02, a section mapping CLI knowledge to this server's tools:
   - `obsidian_read({ file: "Note Name" })` — read a note by wikilink name
   - `obsidian_search({ query: "term", path: "folder/" })` — full-text search
   - `obsidian_daily_read()` — read today's daily note
   - `obsidian_daily_append({ content: "- item" })` — append to daily note
   - `obsidian_create({ name: "New Note", content: "..." })` — create a note
   - `obsidian_tasks({ daily: true })` — list today's tasks
   - `obsidian_tags({ sort: "count" })` — list tags sorted by count
   - `obsidian_properties({ file: "Note Name" })` — read properties
   - `obsidian_property_set({ name: "status", value: "done", file: "Note" })` — set a property
   - `obsidian_backlinks({ file: "Note Name" })` — list backlinks
   - `obsidian_files({ folder: "Projects/" })` — list files
   - `obsidian_recents()` — recently opened files
   - `obsidian({ command: "help search" })` — run any CLI command directly

    - `obsidian_property_set({ name: "tags", value: "project, active", file: "Note" })` to set frontmatter properties
